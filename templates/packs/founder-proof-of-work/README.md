# Founder Proof of Work

Use this pack to keep a local, reviewable record of progress before turning it
into updates, decks, applications, or a personal proof-of-work site.

The workflow is intentionally local-first:

- Log shipped work and user evidence in `log.md`.
- Promote only reviewed material into `index.md`.
- Draft updates from `templates/`.
- Place generated decks, pages, and reports under `outputs/`.
- Keep private facts private until `policies/publish-rules.md` says they are safe.

Suggested cadence:

1. Add raw notes daily.
2. Summarize weekly.
3. Mark evidence as reviewed.
4. Export or publish only reviewed sections.

Suggested future sources, after connector use cases are reviewed:

- GitHub for shipped work and code review evidence.
- Notion for product notes and user research.
- Calendar or CRM for user conversations.
